## Descripción del proyecto

Este proyecto tiene como objetivo comparar dos enfoques de programación para resolver un mismo problema: el registro y visualización de información de mascotas.

Se desarrollaron dos programas en Python:

### 1. Programación Tradicional

Ubicado en la carpeta `programacion_tradicional`.

Este programa utiliza variables, funciones y entrada de datos por teclado para registrar y mostrar la información de una mascota. No se emplean clases ni objetos.

**Datos registrados:**

* Nombre
* Especie
* Edad

### 2. Programación Orientada a Objetos

Ubicado en la carpeta `programacion_poo`.

Este programa utiliza una clase llamada `Mascota` con atributos y métodos para representar el comportamiento y las características de una mascota.

**Atributos:**

* nombre
* especie
* edad

**Métodos:**

* mostrar_informacion()
* hacer_sonido()

Se crean dos objetos de la clase `Mascota` y se ejecutan sus métodos para mostrar la información y el sonido correspondiente.

## Reflexión

Durante el desarrollo de esta actividad se pudo observar que la Programación Tradicional organiza la solución mediante funciones y variables independientes, siendo adecuada para programas pequeños y sencillos.

Por otro lado, la Programación Orientada a Objetos permite agrupar datos y comportamientos relacionados dentro de una misma clase, facilitando la organización, reutilización y mantenimiento del código. Además, el uso de objetos representa de manera más natural los elementos del problema real.

En conclusión, la Programación Orientada a Objetos ofrece una estructura más escalable y ordenada para el desarrollo de aplicaciones, especialmente cuando los proyectos aumentan en tamaño y complejidad.
